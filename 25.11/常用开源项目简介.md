# 介绍一些Java后端学习中可能遇到的比较基础的项目

## 一. Redis

### 1.什么是Redis
Redis 是一个开源的、基于内存的、可持久化的键值对存储系统。它通常被用作数据库、缓存和消息中间件。<br/>
* 开源意味着我们可以找到Redis的源码。<br/>
* 基于内存意味着Redis可以直接访问存储在内存（RAM）中的数据，读写无需磁盘I/O，因此性能非常高，通常能达到每秒数十万次的读写。<br/>
* 可持久化意味着Redis可以将内存中的数据异步保存到硬盘上，防止服务器断电或重启时数据丢失。<br/>
### 2.Redis的数据类型
Redis以键值对(key:value)的形式存储数据，而value支持非常丰富的数据类型。<br/>
* 包括字符串String，哈希Hash，列表List，集合Set，有序集合Sorted Set以及其他更复杂的类型。<br/>
### 3.为什么不怎么用Memcached了?
Memcached也是一个基于内存的键值对存储系统。但它不支持持久化操作、value支持的数据类型单一（只有字符串String类型），所以现在很少用Memcached了。<br/>


## 二.Maven
### 1.什么是Maven
一句话概括，Maven是个项目管理和构建的自动化工具。<br/>
java的源码通常会打包在jar包中，我们为了使用某些特别的方法，往往需要去满世界地找它所属的jar包，除此之外，还必须处理传递依赖和版本冲突两个问题。<br/>
* 传递依赖，即A.jar依赖B.jar，如果要使用A.jar，必须也导入B.jar。<br/>
* 版本冲突，我们在两个地方找到的A.jar和B.jar，如果是两个互不兼容的版本，那么我们是无法正常使用的。<br/>

基于此，Maven完美解决了这几个痛点。<br/>
* 项目管理，一方面来讲，我们可以通过pom.xml文件来指定所需jar包的名称和版本，Maven会自动到中心仓库下载该jar包及相关依赖；另一方面，Maven规定了一个标准化的项目结构，<br/>

```
├── src
│   ├── main
│   │   ├── java       <-- Java 源代码
│   │   └── resources  <-- 配置文件 (.xml, .properties)
│   └── test
│       ├── java       <-- 测试代码
│       └── resources  <-- 测试用的配置文件
├── target             <-- Maven 编译打包后生成的文件
└── pom.xml            <-- 项目的依赖和配置
```
* 自动化工具，Maven预设了一套完善的生命周期命令。只需要输入简单的命令，它就会自动执行一系列流程：编译、测试、打包、部署。<br/>
```
mvn compile;
mvn test;
mvn package;
mvn install;
```

## 三.Spring
### 1.什么是Spring
Spring是一个开源框架，提供了控制反转IOC和面向切面编程AOP两个核心功能。<br/>
* IOC/DI，如果类A需要用到类B，控制正转，即传统做法是在类A中new一个B的对象，对象的控制权在类A；而控制反转则将对象控制权移交给Spring容器，然后有Spring容器将对象B注入到类A中。<br/>
* AOP，举个例子，有一个通用的日志相关的一段代码，如果我们想要输出日志，就要在源码上加入这些通用、业务无关的代码。Spring选择创建一个日志切面，想要哪个部分输出日志，把这个日志切面切入到对应位置的代码处即可。<br/>
* 框架，意味着我们Spring已经为我们构建好了标准的开发结构，我们只需要考虑业务而无需“重新造一个轮子”。Spring框架并不是一个单一的框架，其包含的各种模块可以帮助我们简化开发流程。<br/>

## 四.SpringMVC
### 1.什么是MVC
MVC（Model-View-Controller）模式是一种广泛应用于软件开发中的架构模式，特别是在用户界面的设计中。<br/>
它将应用程序分为三个核心部分：模型（Model）、视图（View）和控制器（Controller），实现了关注点的分离，即将数据的管理、用户界面和控制逻辑分离开来。<br/>
* 模型（Model）
模型是MVC架构中的核心，负责数据和业务逻辑。它管理应用程序的数据结构，处理业务规则、数据的保存和读取。模型与视图和控制器相互独立，不直接与用户界面相关联。当模型的数据发生变化时，它会通知视图，视图可能需要更新用户界面。<br/>
* 视图（View）
视图负责展示数据，即模型中的数据。它将模型的数据以图形界面的形式展示给用户，但不包含业务逻辑。视图仅仅展示数据，并将用户命令（如按钮点击）传递给控制器处理。<br/>
* 控制器（Controller）
控制器作为模型和视图之间的中介，负责接收用户的输入，并调用模型和视图去完成用户的请求。控制器会处理用户的输入（例如，鼠标点击、键盘输入），并将这些输入转换为对模型和视图的命令。控制器可以改变模型的状态，也可以改变视图的展示。<br/>

以上摘自[链接](https://blog.csdn.net/liitdar/article/details/86685880)<br/>

### 2.什么是SpringMVC
一句话来说，SpringMVC是Spring框架中，基于MVC设计模式的Web框架。<br/>
通过Spring MVC的核心流程（请求→DispatcherServlet→Controller→Service→视图），我们可以很规范地完成Web后端应用程序的开发。<br/>

## 五.Spring Boot
### 1.什么是Spring Boot
通过自动配置和起步依赖，极大简化Spring开发的脚手架工具。<br/>
哦齁齁，绝对的重量级来了。<br/>
如果说，我们在开发中要用到的配置和依赖绝大部分都相同，那么是不是就可以将大部分相同的配置和依赖都自动装载起来，让别人使用？<br/>
* 自动配置，即Spring容器自动装载了一些常用的Bean类，我们要用的时候直接注入即可，也就是IOC。<br/>
* 起步依赖，举个例子，既然开发Web所用的依赖都差不多，那就把这些常用的依赖打包到起步依赖中。引入起步依赖，就相当于引入了所有这些依赖。<br/>

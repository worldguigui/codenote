# Java基础--字符串

## 一.概述
Java内没有内置的字符串类型，由标准Java类库提供了预定义类。

有两种方式创建Java字符串对象：

引号直接创建字符串字面量，或通过new String()创建。

```java
String s1 = "abc";
// 下面这个语句创建了几个字符串对象？答案是：两个！
String s2 = new String("abc");
```
---
## 二.共享字符串
在Java设计理念中，认为共享字符串比起修改字符串来讲会更加高效。

所以对于一个字符串来讲，我们无法去修改它的某一位字符，只能通过剪切拼接的方式得到一个新字符串。

字符串字面量存储在字符串变量池中，当创建了一个新的字符串时，JVM会在池中寻找这个字符串是否已经存在，存在则将变量引用指向它，不存在则创建。

所以，对于相同的字符串字面量，它们的地址是相同的。

这就实现了字符串的共享。

```java
String s1 = "abc";
String s2 = "abc";
System.out.println(s1 == s2);
// true
```

---
## 三.字符串为什么不可变
String是Java中的标准类库提供的预定义类，它的底层用final修饰了存储数据的变量。

String类中还有字段用来记录字符串的哈希值（在第一次计算时更新），这在以字符串在为键的场景中带来了较好的性能。

```java
private final byte[] value;

private int hash; // Default to 0
```

---
## 四.字符串在堆中的位置
常量池是JVM中用于存储常量的一种机制，包括Class文件常量池和运行时常量池，而字符串常量池是其中专门用于缓存字符串对象的一部分。

从JDK7开始，字符串常量池从常量池中抽离，位于堆中。

在这之前，常量池位于堆中的永久代/方法区中，为了与其他的JVM匹配（其他的JVM没有永久代，HOT SPOT独有）和避免OOM，之后的版本移除了永久代。

而是在系统内存中分配了一块元空间，用来代替永久代。（元空间的大小与系统内存直接关联，当然也可以手动设置）

只有字符串字面量或调用intern方法的字符串才会进入字符串常量池，而通过new创建的字符串对象本身是在堆中。

```java
// 通过new创建了字符串对象，并通过方法intern()将其引用放进了字符串常量池
// intern()会返回常量池中的引用。如果常量池中没有该字符串，则会将当前字符串对象的引用存入常量池
String s1 = new String("abc").intern();
// s2直接指向池中已经存在"abc"
String s2 = "abc";
System.out.println(s1 == s2);
// true
```

## 五.常量池
**在编译阶段--class文件常量池**

编译器将Java源代码编译成.class字节码文件后，

字符串字面量（"abc"）、类名方法名、字段名、符号引用等会被放在池中。

**在JVM运行阶段--运行时常量池**

类加载器将字节码文件加载到内存中，系统为JVM分配内存。

字符串进入到堆中的池，其他进入到元空间。

### 六.StringBuilder和StringBuffer
StringBuilder底层是一个可扩容的字符数组（JDK9后为byte[]），通过原地修改和自动扩容机制避免创建中间字符串对象，从而提高字符串拼接性能。

原地修改，就是将字符数组拷贝到sb底层的字符数组中。

扩容后的数组容量是扩容前的两倍。
```java
String s = "a";
s += "b";
s += "c";
// 在拼接过程中，字符串常量池中依次创建了 a b ab c abc 吗？并不是！
// 编译器会优化类似的单次拼接，
// 在每次 += 拼接时，编译器会将其转换为：
// new StringBuilder().append(...).append(...).toString()
// 拼接次数少时，带来的开销较小。
// 若是拼接1000次，就要创建1000次sb，在堆中创建1000次对象，进行1000次数组拷贝。

因此每次拼接都会创建一个新的 StringBuilder 对象，
并生成一个新的 String 对象。

StringBuilder sb = new StringBuilder();
sb.append("a");
sb.append("b");
sb.append("c");
String s = sb.toString();
// 在拼接过程中，字符串常量池中依次创建了 a b c，堆中创建了字符串对象 "abc"
```

StringBuffer和sb的效果大致相同，都可以提高字符串拼接性能。

但和sb唯一不同的是，sf底层通过synchronized关键字，保证了拼接字符串时的线程安全。

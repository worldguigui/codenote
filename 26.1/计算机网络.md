# 计算机网络

## 一.概述
### 1.定义 
计算机网络是地理位置不同的、具有独立功能的多台计算机及其外部设备，通过通信线路和交换设备连接起来，按照某种协议进行数据通信，实现资源共享和信息传递的系统。</br>

资源共享、信息交换是计算机网络的主要目标。</br>

### 2.物理组成
计算机网络由什么组成？</br>
端系统 + 通信链路 + 交换设备。</br>

#### 基本概念
1.网络边缘：用户直接接触到的部分，如主机和应用。</br>
2.核心部分：由大量网络和连接这些网络的路由器组成。</br>
3.端系统(End System)：也叫主机(Host)，位于网络边缘，产生或消费数据的设备（主机），如你的电脑、手机、服务器。</br>
4.通信链路：数据传输的介质。</br>
5.交换设备：连接多个通信链路，并根据特定规则接收数据，然后选择一条输出链路将其转发出去的专用硬件设备。包含路由器和交换机。</br>
6.路由器：连接不同网络，基于网络层地址(主要是IP地址)进行路径的选择和转发。</br>
7.交换机：连接同一网络内的端系统，基于数据链路层地址（通常是MAC地址）进行转发。</br>
8.通信子网：从逻辑功能角度来看，计算机网络分为资源子网和通信子网。通信子网负责将信息从主机传到另一台主机。</br>
9.资源子网：使用信息资源的部分，如主机、服务器。</br>
10.ISP：Internet Service Provider，即互联网服务提供商，比如说中国移动、电信等。</br>

#### 端系统的两种连接方式
1.客户(Client)和服务器(Server)：</br>
客户是服务的请求方，服务器是服务的提供方。</br>

2.对等连接(peer-to-peer，P2P)：</br>
对于对等连接的两个主机，它们都可以请求和提供服务。</br>

### 3.性能衡量指标
1.速率 (Data Rate)：连接在计算机网络上的主机在数字信道上传输数据的绝对速度，单位是比特每秒 (bps)。</br>
2.带宽 (Bandwidth)：通信链路本身在单位时间内能够通过的最高数据率，单位是比特每秒 (bps)。</br>
3.吞吐量 (Throughput)：在单位时间内，通过网络中特定链路或通道实际成功传输的数据量。这是一个实际测量值，而非理论值。单位是比特每秒 (bps)。</br>
4.丢包率 (Packet Loss Rate)：在数据传输过程中，丢失的数据包数量占已发送数据包总数的百分比。</br>
5.时延 (Delay / Latency)：数据（一个比特或一个数据包）从网络的一端传送到另一端所经历的时间。</br>
总时延 = 发送时延 + 传播时延 + 处理时延 + 排队时延。</br>
发送时延 = 数据包长度 (bits) / 信道带宽 (bps)。</br>
传播时延 = 信道长度 (meters) / 电磁波在介质中的传播速率 (m/s)。传播速率在真空中为光速，在光纤中约为光速的 2/3。</br>
处理时延：路由器或主机在接收到数据包后，进行存储、检查首部、决定转发路径等操作所花费的时间。</br>
排队时延：数据包在路由器或交换机的输入/输出队列中等待被处理或等待链路空闲的时间。当网络拥塞时，排队时延会显著增加。</br>

### 4.分类
1.按通信介质：有线网络、无线网络。</br>
2.按传输速度：低速网络、高速网络。</br>
3.按传播技术：广播式网络、点到点式网络。</br>
4.按分布范围：广域网WAN（一个国家、大洲）、城域网MAN、局域网LAN、个域网PAN。</br>
5.按使用者：公用网络、专用网络（教育、军队、银行）。</br>
6.按拓扑结构：总线型、星型、环型、网状型。</br>
7.数据交换技术：电路交换网络、报文交换网络、分组交换网络。</br>

电路交换：传统电话网的思维方式。先建立一条独占的物理/逻辑通路，再通话。效率低，专线专用。</br>
保温交换：将整个消息作为一个整体，在每个节点存储、再转发。</br>
分组交换：将数据分割成一个个带地址的小数据包，每个包独立传输，在目的地重组。</br>

### 5.层次结构和协议
计算机网络中，层和协议的集合被称为计算机网络体系结构。</br>

层（Layer）：计算机网络将整个过程分层，每层只负责基于下层服务，向上层提供服务，不关心其他层接口的具体实现。</br>

协议（Protocol）：定义网络实体间发送和接收报文的格式、顺序以及当传送和接收消息时应采取的行动。（语义、语法和时序）</br>

#### OSI七层模型
国际标准化组织（ISO）正式发布的理想模型，由七层构成。

#### TCP/IP参考模型
TCP/IP协议是ARPANET和因特网上使用的一组协议，是一个协议簇。
TCP/IP起源于美国国防部高级研究规划署(DARPA)的一项研究计划——实现若干台主机的相互通信。


层次	名称	核心职责	关键协议	关键设备
5	应用层	融合了OSI不常用的应用、表示、会话三层功能，直接面向用户进程。	HTTP (Web)、DNS (解析)、SMTP (邮件)、FTP (文件)	主机
4	传输层	与OSI传输层完全对应。提供端到端通信。	TCP (可靠连接)、UDP (高效无连接)	主机
3	网络层	与OSI网络层完全对应。负责IP寻址和路由。	IP (核心)、ICMP (控制消息)、路由协议	路由器
2	数据链路层	融合了OSI的数据链路层功能。负责局域网内帧传输。	以太网、Wi-Fi、PPP	交换机、网桥
1	物理层	与OSI物理层对应。双绞线、光纤、无线电波	集线器、中继器、网卡

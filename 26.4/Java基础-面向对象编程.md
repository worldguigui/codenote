# Java基础-面向对象编程

## 一.引言
封装、继承、多态是面向对象编程（Object-Oriented Programming，OOP）的三大特性。

Java是一门典型的面向对象的语言。

## 二.封装
将属性和方法封装在类的内部，对外隐藏内部细节，只暴露方法给外部调用。

通过权限控制符（private、public、protected）来控制外部对内的可见程度。

private：仅类内部可以访问。

default（默认）：同一包的类可以访问。

protected：子类和同一包的类可以访问。

public：公开，允许所有类访问。

## 三.封装的底层细节--如何控制访问权限？
Java中的封装通过访问修饰符（private、protected、public）来实现，其底层依赖于编译期和运行期两层机制。

在编译期，编译器会进行语法检查，确保代码不会非法访问私有成员。

在运行期，JVM会根据字节码中的访问标志（access_flag）进行权限校验，如果访问不合法会抛出异常。

需要注意的是，通过反射可以调用 setAccessible(true) 来关闭访问检查，从而访问私有成员，但这属于对封装机制的绕过，而不是JVM本身不做校验。

## 四.继承
继承表示子类和父类有 is-a 关系，实现代码复用和扩展，通过 extends 实现。

子类会继承父类所有非私有的属性和方法（不包括构造方法）。

Java中，只支持单继承，和链式继承。不允许多继承。

```java
// ×
class A extends B,C{}
// √
class A extends B{}
class B extends C{}
```

### 1.this和super
this指向当前对象，super是访问父类成员的一种语法关键字

---
this() 规则：

可以出现在任何构造中，但必须在第一行。

不能和super同时出现。（因为不可能让它们同时在同一行）

创建对象时，默认调用无参构造；若类中没有构造，使用默认的无参构造。

---

super() 规则：

可以出现在任何构造中，但必须在第一行。

如果子类构造方法没有写 super()，编译器会自动加 super()（调用父类无参构造）

如果父类没有无参构造，子类必须手动调用 super(参数)

### 2.方法重载和重写
方法重载，允许名称相同、但参数列表不同的方法。

---

方法重写：

子类可以修改继承自父类的方法的逻辑，名称和参数列表要保持不变，返回值类型必须相同（或是其子类）

可以选择用注解 @Override 标识。

重写的方法的可见程度不能比父类更严格。

不能抛出父类没有声明的检查型异常。

## 五.继承的底层细节
### 1.对象内存布局
当创建一个子类对象时，JVM 会在堆中连续分配一块内存，包含：对象头 + 父类实例数据 + 子类实例数据。

---
在这块内存中：

父类和子类的数据在同一块连续内存中

子类对象包含完整的父类数据

访问父类字段时，通过固定偏移量直接定位

---

字段访问在编译期确定访问的是哪个类的字段，在类加载阶段确定字段在对象中的偏移量，属于静态绑定。

### 2.虚方法表执行流程

通过 a 找到堆中的 Dog 对象

从对象头中获取类元数据指针

从类元数据中找到虚方法表
 
根据方法在虚方法表中的索引查找对应的方法入口

跳转到 Dog.eat() 执行

```java
class Animal {
    public void eat() { System.out.println("动物吃"); }
    public void sleep() { System.out.println("动物睡"); }
}

class Dog extends Animal {
    @Override
    public void eat() { System.out.println("狗吃骨头"); }
    public void bark() { System.out.println("汪汪"); }
}

Animal 类的虚方法表：
┌─────────────────────────────────────────┐
│ 索引 │ 方法        │ 入口地址            │
├─────────────────────────────────────────┤
│ 0    │ eat()       │ Animal.eat() 地址   │
│ 1    │ sleep()     │ Animal.sleep() 地址 │
│ 2    │ toString()  │ Object.toString()   │
└─────────────────────────────────────────┘

Dog 类的虚方法表：
┌─────────────────────────────────────────┐
│ 索引 │ 方法        │ 入口地址            │
├─────────────────────────────────────────┤
│ 0    │ eat()       │ Dog.eat() 地址 ← 覆盖│
│ 1    │ sleep()     │ Animal.sleep() 地址 │
│ 2    │ toString()  │ Object.toString()   │
│ 3    │ bark()      │ Dog.bark() 地址 ← 新增│
└─────────────────────────────────────────┘
```
## 六.多态
同一类型的对象，可以表现出不同形态。

### 1.多态的前提
有继承/实现关系

有方法的重写

有父类引用指向子类对象

### 2.多态中调用成员的特点
调用成员变量，编译看左边，运行也看左边：

编译Java代码时，先看左边的父类有没有这个变量，有则编译成功，没有则失败。

运行时，实际获取的是左边父类中成员变量的值。

---

调用成员方法，编译看左边，运行看右边

编译Java代码时，先看左边的父类有没有这个方法，有则编译成功，没有则失败。

运行时，实际执行的是子类中的方法。

```java
class Human{
    public String id;
    public Human() { id = "Parent"; }
    public void eat() { System.out.println("human is eating") }
}

class Man extends Human{
    public String id;
    public Man() { id = "Child"; }
    @Override public void eat() { System.out.println("man is eating") }
}


Human h = new Man();
System.out.println(h.id);
h.eat();

// Parent
// man is eating
```

## 六.多态的底层细节-虚方法表
在Java中，多态的实现依赖于JVM的动态绑定机制。

当类被加载时，会先加载父类，再加载子类，并在方法区（JDK8为Metaspace）中存储类的元数据。

JVM会为类构建虚方法表（vtable），该表用于存储方法的方法指针，只有可被重写的虚方法（非static、非private、非final）会在虚方法表中。

子类在构建方法表时会继承父类的方法结构，如果发生方法重写，则会替换对应方法的入口地址。

随着main方法入栈，栈帧中先是创建了父类的引用，并指向堆中创建的子类对象。

字段访问是静态绑定的，在编译期根据引用类型确定，不参与多态，在h.id访问时，只会找到父类的字段。

在方法调用时，JVM通过对象头中的类型指针定位到实际类，再通过虚方法表查找具体方法，实现运行时动态绑定。

这样在h.eat()执行时，查找到了虚方法表上重写后的eat方法。
